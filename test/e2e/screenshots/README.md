# Puppeteer Test Screenshots

This directory contains screenshots generated by Puppeteer automated tests.

## Directory Structure

```
screenshots/
├── selection-box/           # SelectionBoxController tests
│   ├── success/             # Passing test screenshots
│   │   ├── test1_basic_rendering.png
│   │   ├── test2_configuration.png
│   │   ├── test4_toggle.png
│   │   └── test5_debug_info.png
│   ├── failure/             # Failing test screenshots
│   │   └── test3_callbacks.png
│   └── *.png                # Step-by-step debug screenshots
└── [future-test-category]/  # Other test categories follow same pattern
    ├── success/
    ├── failure/
    └── *.png
```

## Naming Convention

### Test Result Screenshots
- **Location**: `{category}/success/` or `{category}/failure/`
- **Format**: `test{number}_{testname}.png`
- **Purpose**: Final result screenshot for each test showing pass/fail state

### Step Screenshots
- **Location**: `{category}/` (root of test category)
- **Format**: `{number}-{step-name}.png`
- **Purpose**: Debugging screenshots showing intermediate steps during test execution

## Current Test Categories

### selection-box
Tests for enhanced SelectionBoxController features:
- Configuration API
- Callback system
- Enable/disable toggle
- Selection rendering
- Debug info API

**Test Results**: 4/5 Passing (80%)

## Usage

Screenshots are automatically generated when running tests:
```bash
node test/puppeteer/selection-box.test.js
```

## Maintenance

- Keep success screenshots to track passing tests over time
- Review failure screenshots to debug test failures
- Clean up old step screenshots periodically (they regenerate on each run)
- Archive screenshots when making major changes to features
